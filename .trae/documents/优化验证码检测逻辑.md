# 解决验证码检测失效的最终方案

根据您的反馈和我的深入分析，之前的检测方案失效主要有两个原因：
1.  **Iframe 过滤过严**：之前的代码只检查 URL 包含 `captcha` 的 Frame，但抖音的验证码 Iframe 地址经常变化（有时甚至是 `about:blank` 动态加载），导致漏检。
2.  **CSS 选择器失效**：抖音的前端代码更新频繁，依赖特定的 CSS 类名（如 `.captcha_verify_container`）非常脆弱。

我将采用**“文本特征全域扫描”**的方案来彻底解决这个问题。

## 核心改进逻辑

### 1. 改用“文本匹配”而非“类名匹配”
我们不再去猜验证码容器叫什么名字，而是直接找弹窗上一定会写的字。
根据您的截图，最显眼的特征文本是：**“请完成下列验证后继续”**。
只要页面上（无论是主页面还是任何角落的 Iframe 里）出现了这句话，就百分之百是弹验证码了。

### 2. 移除 Iframe 过滤条件
不再判断 Frame 的 URL 是否包含 `captcha`。我们将对页面上**所有的 Frame** 进行无差别扫描。

### 3. 增加兜底检测
除了文本，保留一组通用的 ID 和 Class 特征作为备选，确保万无一失：
*   `text="请完成下列验证后继续"` (核心)
*   `text="验证码"`
*   `#captcha-verify-image` (图片 ID)
*   `[class*="captcha"]` (模糊匹配)

## 修改计划

我将修改 `liker.py` 中的 `_run_browser` 方法，重写验证码检测模块：

```python
# 伪代码预览
async def check_captcha():
    # 1. 定义特征列表
    indicators = [
        self.page.get_by_text("请完成下列验证后继续"),
        self.page.locator("#captcha-verify-image"),
        self.page.locator(".captcha_verify_container")
    ]
    
    # 2. 检查主页面
    for ind in indicators:
        if await ind.is_visible(): return True
        
    # 3. 检查所有 Frame (无差别扫描)
    for frame in self.page.frames:
        try:
            if await frame.get_by_text("请完成下列验证后继续").is_visible(): return True
            if await frame.locator("#captcha-verify-image").is_visible(): return True
        except: continue
            
    return False
```

这将彻底解决检测不到的问题。确认后我立即执行代码修改。
