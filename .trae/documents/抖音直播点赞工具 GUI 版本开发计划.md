# 抖音直播点赞工具 GUI 升级计划

我将为您把现有的命令行工具改造成一个拥有图形界面（GUI）的桌面软件，并添加您要求的新功能。我们将使用 Python 内置的 **Tkinter** 库来实现界面，这样您不需要安装额外的大型界面库，运行更加轻便。

## 1. 界面设计 (GUI)
我们将创建一个现代化的控制面板，包含以下区域：

*   **基本设置区**：
    *   **直播间链接**：输入框，支持粘贴。
    *   **浏览模式**：勾选框，选择是否“后台静默运行”或“显示浏览器窗口”。
    *   **一键登录**：按钮，点击后弹出扫码登录窗口（调用原 `auth.py` 逻辑）。
*   **点赞参数区**：
    *   **点击频率**：两个输入框，设置“最快”和“最慢”间隔（例如 0.1秒 - 0.5秒）。
    *   **循环模式（新功能）**：
        *   **点赞时长**：输入框（分钟），设定连续点赞多久。
        *   **休息时长**：输入框（分钟），设定休息多久。
        *   *启用/禁用* 开关。
*   **控制与反馈区**：
    *   **开始/停止**：大按钮，控制程序运行。
    *   **运行日志**：滚动文本框，实时显示“正在点赞...”、“已暂停休息...”、以及“直播已结束”等提示。
    *   **状态栏**：显示当前累计点赞次数。

## 2. 新功能实现细节

### 功能一：循环点赞模式 (Cycle Mode)
*   **逻辑**：程序内部维护一个计时器。
*   **工作流**：
    1.  状态：`运行中` -> 持续点赞，直到达到设定的“点赞时长”。
    2.  切换：暂停点赞，进入`休息中`状态。
    3.  等待：倒计时“休息时长”。
    4.  恢复：重新进入`运行中`状态，循环往复。
*   **优势**：这种“工作-休息”的模式更像真人行为，能显著降低账号被风控的风险。

### 功能二：直播结束自动停止
*   **检测机制**：
    *   **文本检测**：自动扫描页面是否出现“直播已结束”、“已下播”等关键词。
    *   **元素检测**：如果直播视频流容器 (`video` 标签) 消失超过 1 分钟，则判定为直播结束。
*   **响应动作**：
    *   立即停止点赞循环。
    *   关闭浏览器进程。
    *   **弹窗提示**：在界面上弹出一个提示框：“检测到直播已结束，任务自动停止。”

## 3. 技术重构
为了实现界面不卡顿（GUI 响应）与后台点赞（自动化任务）同时进行，我将对代码结构进行重大升级：
*   **多线程架构**：
    *   **主线程**：负责显示界面、接收您的点击、更新日志窗口。
    *   **后台线程**：负责运行 `Playwright` 浏览器自动化逻辑。
*   **类封装**：
    *   创建 `DouYinLiker` 类，封装所有浏览器操作，不再使用散乱的函数。

*   `run_gui.bat`: 一键启动脚本。
