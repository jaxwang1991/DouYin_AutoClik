# 添加“手速检测”与“暂停/恢复”功能计划

我将为您在现有的 GUI 版本基础上增加这两个实用的新功能。

## 1. 功能设计

### 功能一：智能冷却（防封控）
*   **触发条件**：自动检测直播间页面是否弹出 **“手速太快了，请休息一下吧~”** 的提示。
*   **响应动作**：
    1.  立即停止点赞。
    2.  界面状态更新为 **“冷却中 (剩余 xxx 秒)”**。
    3.  **倒计时 3 分钟**（180秒）。
    4.  倒计时结束后，自动恢复点赞，并继续检测。

### 功能二：手动暂停/恢复
*   **界面变化**：在“开始任务”和“停止任务”之间，新增 **“暂停”** 和 **“恢复”** 按钮。
*   **逻辑**：
    *   点击 **暂停**：浏览器保持打开，但停止点击动作。状态显示为 **“已暂停 (用户)”**。
    *   点击 **恢复**：立即继续点击。

## 2. 代码修改计划

### 修改 `liker.py` (核心逻辑)
1.  **新增状态**：添加 `PAUSED_BY_USER` (用户暂停) 和 `COOLDOWN` (冷却中) 状态。
2.  **新增控制方法**：添加 `pause()` 和 `resume()` 方法供 GUI 调用。
3.  **完善主循环**：
    *   在循环开头增加对 `manual_pause` 标志的检查。
    *   在循环中增加对 **“手速太快了”** 文本的检测。
    *   如果检测到该文本，进入 180 秒的倒计时循环，期间每秒更新一次 UI 状态（显示倒计时）。

### 修改 `gui.py` (界面)
1.  **新增按钮**：在底部控制栏添加 `btn_pause` 和 `btn_resume`。
2.  **按钮逻辑**：
    *   `start_task` 后：启用“暂停”和“停止”按钮。
    *   点击“暂停”后：禁用“暂停”，启用“恢复”。
    *   点击“恢复”后：禁用“恢复”，启用“暂停”。
3.  **状态反馈**：适配新的状态文本显示（如显示冷却倒计时）。

## 3. 预期效果
*   **更安全**：遇到官方警告自动认怂（休息3分钟），极大降低封号风险。
*   **更灵活**：您可以随时暂停程序去回个消息，然后回来继续点赞，不用每次都关闭重开。
